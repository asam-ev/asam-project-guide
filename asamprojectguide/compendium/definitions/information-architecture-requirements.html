Requirements of the information architecture

Requirement categories

  • build [B]

  • syntax [X]

  • structure [S]

  • technical [T]

  • writing rules [W]

  • content [C]

  • document [D]

Requirements list

Table 1. Requirements List
Type ID Short Description Status Ref

[B]

❔B-1: No fixed versioning in any file

✅B-2: List of figures is created during the build process

The List of tables and list of Figures will be created automatically

Link

✅B-3: List of tables is created during the build process

The List of tables and list of Figures will be created automatically

Link

🆕B-4: General spellcheck is passed

✅B-5: Bibliography is created during the build process

✅B-6: The files of the deliverables are named during the build process

🆕B-7: Checks can be customized based on project

🆕B-8: Spellchecker can be deactivated for single words

[X]

❔X-1: Headings use correct syntax

❔X-2: Headings use sentence capitalization

✅X-3: Every image shall have an unique anchor starting with "fig-"

✅X-4: Every table shall have an unique anchor starting with "tab-"

The structure of every standad should follow the given structure in the ASAM Standard Template Repository

Link

✅X-5: Every topic shall have an unique anchor starting with "top-"

✅X-6: Every code block shall have an unique anchor starting with "code-"

✅X-7: Every other referencably block shall have an unique anchor

❔X-8: Anchors shall be defined in the form

🆕X-9: Max line length for code blocks

🆕X-10: No Trailing whitespaces

🆕X-11: Using Tables to structure Text where needed

Tables can help to structure text in ASCII Doc, these table should not be given a header and therefore will not be visible in the list of tables. NOTE: use grid=none, frame=none to remove the frame and lines

✅X-12: Sections may have an unique anchor starting with "sec-", when cross-references to sections are needed

❔X-13: For references on images and tables `:xrefstyle: short` shall be used, for example Figure 1 and Table 1

❔X-14: The complete Table of contents shall be defined in the main.adoc file using the leveloffset attribute

❔X-15: Acronyms and proper names in headings are written according to the specific rules in writing

❔X-16: Requirements shall use modal verb according to ISO guidelines

[S]

🆕S-1: Documents are named after the ASAM Document Naming convention

🆕S-2: ASAM Standard Chapter structure

🆕S-3: There are now empty chapters

🆕S-4: The folders which contain the content for the deliverables shall be named _standard_ for standards or _user_guide_ for user guides

🆕S-5: The file of the main document for the content of the deliverables shall be named main.adoc

🆕S-6: Numbering of subfolders & documents

Standard folders and documents therein are numbered in order they occur using a 2 digit numeral (00-NN)

🆕S-7: The technical content of a standard in chapter 6

[T]

✅T-1: Document Extention is .adoc

All documents shall have .adoc as file extention

Link

🆕T-2: Formulas use latexmath

🆕T-3: Graphics use the provided graphics toolkit

All graphics are created using the provided assets. They are created in drawio or plantUML.

🆕T-4: No unused images in repo

🆕T-5: Diagrams use .drawio.svg extension

🆕T-6: There are no unused (not included) chapters in the repository

When a line in a code block is to long, a ugly line break is done. Therefore, there should be a maximum line length.

🆕T-7: All filenames shall be lowercase

🆕T-8: The first commit in the repo shall be tagged

🆕T-9: The project README shall contain the roadmap of the project

[W]

🆕W-1: Names of standards correctly used

✅W-2: One sentence per line

Each sentence is in its own line.

Link

🆕W-3: Every document shall have a disclaimer

🆕W-4: The disclaimer shall be included using the template which is present in the templates repository

🆕W-5: Every figure shall have a description underneath the figure

🆕W-6: Oxford commas are not used

🆕W-7: Fragements shall contain meta data using the 'keywords' attribute

[C]

🆕C-1: Repository shall contain LICENSE file

🆕C-2: Repository shall contain overview roadmap, responsible GTM, lead, WG

{C-5}

The specific technical content of standard should be in chapter 6 and its subfolders, once this is unified any reader will know, to look in chapter 6 for technical specifications.

[D]

🆕D-2: Included images are shown

🆕D-3: Links are functional
