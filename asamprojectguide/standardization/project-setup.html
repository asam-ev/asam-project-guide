Proposal setup

Project Proposal setup

GTM

For information on the proposal phase, see The Proposal phase.

TODO: Add image with highlighted step here.

Context

The basis for collaboration at ASAM is GitLab. In order for project members and the project lead to work on their respective project, a new repository has to be created.

Result

A new proposal repository has been created and the link been provided to all current project members.

Prerequisites

  • The idea has been proposed and approved in the Ideation Phase.

How

  1. In case of a new standard, create a new subgroup in the group Simulation with the target name of the standard.

    If this is an evolution of an existing standard, use the existing subgroup instead

  2. Fork the latest version of the proposal template repository into this subgroup.

  3. Rename the forked project to <THIS_STANDARD> Proposal, where <THIS_STANDARD> is the target name of the standard.

  4. Add all current project members and the project lead (ideagiver) to the subgroup.

  5. Send a mail with the link to the new project to the group, informing them that the repository is now ready for use.
