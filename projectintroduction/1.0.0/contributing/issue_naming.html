Issue naming conventions

Convention

group-name: issue-title

General guidelines

  1. Applies to all issues EXCEPT during Proposal phase

  2. If applicable, use the group-name (uppercase)

Group names

The group names are defined by the project. The table below shows an example.

Table 1. Group names for the OSI project
Name Description

sm

Sensor modeling

tp

Traffic Participants

ha

Harmonization

pp

Performance & packaging

ge

Any other, non-group-specific developments

Issue title

  1. Should be descriptive and make the topic clear, i.e. not "Update chapter", but rather "Pedestrian to pedestrian interaction"

Examples

TP: Lane subtype
PP: Support for flatbuffer
GE: Index specification of trace files
